1. Computer Systems, CPU & Assembly

The Central Processing Unit (CPU) executes instructions using the Fetch-Decode-Execute (FDE) cycle.

  • Computer Specs: CPU performance is affected by Clock Speed (Hz), Number of Cores, and Cache Size.
  • System Software: Controls the hardware. Includes the Operating System (OS) and utility programs.
  • Assembly Language: A low-level programming language that uses mnemonics (like ADD, SUB). It requires an Assembler to translate it into machine code (binary) before the CPU can execute it.

2. Main Memory & Secondary Storage

Systems rely on memory to balance speed against permanent storage:

  • RAM (Volatile): Main memory. Stores the OS and currently open programs. Loses data when powered off.
  • ROM (Non-Volatile): Read-only memory. Stores the boot-up instructions (BIOS).

Selecting a Storage Device:

  • Optical (CD/DVD): Lasers read physical pits/lands. Highly portable, low capacity, easily scratched.
  • Magnetic (HDD/Tape): Spinning magnetic platters. High capacity, cheap per GB, slow, moving parts break easily.
  • Solid-State (SSD): Electrical flash memory. Extremely fast, durable (no moving parts), expensive.

3. Logic Gates & Flowcharts

Boolean Logic Gates

  • AND Gate: Output is 1 only if both inputs are 1.
  • OR Gate: Output is 1 if at least one input is 1.
  • NOT Gate: Inverts the input (1 becomes 0).

Flowchart Symbols

Oval (Terminal): Start / Stop
Parallelogram: Input / Output
Rectangle: Process / Calculation
Diamond: Decision (Yes/No branches)

4. Computational Thinking & Algorithms

  • Abstraction: Removing unnecessary details from a problem to focus on the essential features.
  • Decomposition: Breaking a complex problem down into smaller, more manageable sub-problems.
  • Algorithmic Thinking: Creating a logical, step-by-step sequence of instructions to solve a problem.
  • Tracing Algorithms: Using trace tables to track how the values of variables change line-by-line during program execution.

5. Searching & Sorting Algorithms

Searching:

  • Linear Search: Checks each item one by one from the start. Works on unordered lists. Slow for large datasets.
  • Binary Search: Finds the middle item, discards the half where the target cannot be. List MUST be ordered first. Very fast.

Sorting:

  • Bubble Sort: Compares adjacent pairs and swaps them if they are in the wrong order. One full "pass" bubbles the largest item to the end. Very slow for large lists.

6. Number Bases & Binary Arithmetic

Binary Arithmetic

  • Addition Rules: 1+1 = 10 (0, carry 1), 1+1+1 = 11 (1, carry 1).
  • Subtraction: AQA often tests this using Two's Complement addition.
  • Binary Shifts: Left shift multiplies by 2. Right shift divides by 2.
  • Signed Binary: Uses the Most Significant Bit (MSB) to represent positive (0) or negative (1).

Hexadecimal (Base-16)

Uses digits 0-9 and letters A-F (where A=10, F=15). Used by humans because it is shorter and easier to read than long binary strings.

Hex to Binary: Split the hex into digits. Convert each digit into a 4-bit binary nibble.

7. Representing Text, Images & Sound

Text

  • ASCII: Uses 7 bits. Can represent 128 characters (English).
  • Unicode: Uses 16 to 32 bits. Can represent every language character globally, plus emojis, but takes up more file space.

Images (Bitmaps)

  • Made of tiny dots called pixels.
  • Color Depth: Number of bits used per pixel. Higher depth = more colors but larger file size.
  • Resolution: Concentration of pixels.
  • File Size = Width × Height × Color Depth

Sound

  • Analogue sound waves are sampled at regular intervals to create digital data.
  • Sample Rate: Number of samples taken per second (Hz).
  • Bit Depth: Number of bits used to record each sample.
  • Size = Sample Rate × Duration (s) × Bit Depth

8. Storage Measurements & Compression

Measurements: AQA strictly uses decimal calculations (divide by 1,000).

Bits → (÷8) → Bytes → (÷1000) → kB → (÷1000) → MB → (÷1000) → GB

Compression:

Reduces file sizes to save storage space and decrease network download times.

  • Lossless Compression: Compresses the file without losing any original data. When uncompressed, the file is exactly the same as the original (e.g., text documents, code files).
  • Lossy Compression: Permanently deletes unnoticeable data to heavily reduce file size. Once compressed, the original quality cannot be restored (e.g., JPEG images, MP3 audio).
