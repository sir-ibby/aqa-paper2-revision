1. CPU Architecture & The FDE Cycle

The Central Processing Unit (CPU) executes instructions using the Fetch-Decode-Execute cycle:

  • Fetch: The next instruction is retrieved from main memory (RAM).
  • Decode: The Control Unit (CU) splits and translates the instruction.
  • Execute: The Arithmetic Logic Unit (ALU) or system hardware carries out the processing action.

Core Hardware Components:

  • ALU: Executes all calculations and logic operations (AND, OR, NOT).
  • Control Unit (CU): Manages the timing and structural data control flow inside the CPU.
  • Registers: Super-fast internal storage slots used to hold current processing states.

2. Memory Architecture & Volatility

Systems rely on various tiers of memory to balance processing speed against persistence:

  • RAM (Random Access Memory): Volatile main memory. Stores open operating system processes and application runtime states. Loses contents on power loss.
  • ROM (Read-Only Memory): Non-Volatile startup memory. Stores immutable factory initialization boot instructions (BIOS).
  • Secondary Storage: Permanent non-volatile storage necessary for archiving user system programs.

The 3 Storage Media Types:

  • Magnetic (HDD): Mechanical spinning discs. High capacity and lower financial cost, but slower read speeds and fragile.
  • Solid-State (SSD): Electrical flash memory chips. Extremely high speed, energy efficient, durable, but premium cost.
  • Optical (DVD): Laser reflection pitting. High physical portability, but small capacity boundaries and prone to scratching.

3. Data Representation: AQA Storage Unit Math

CRITICAL RULE: Under the AQA specification, calculations scale strictly using decimal prefixes (multiples of 1000). Do not divide or multiply by 1024.

Bits → (÷8) → Bytes → (÷1000) → Kilobytes (kB) → (÷1000) → Megabytes (MB) → (÷1000) → Gigabytes (GB)

Binary Bit Shifts:

  • Left Shift: Shifting bit placements by n places multiplies the equivalent target number evaluation by 2n.
  • Right Shift: Shifting bit placements by n places divides the equivalent target number evaluation by 2n (completely truncate/ignore any fractional remainder values).

4. Computer Network Design & Topologies

A network setup dictates how interconnected devices physically or logically exchange parameters:

  • Star Topology: All network endpoints converge upon a singular central device (Switch/Router).
    • Pros: Isolated link disruptions do not drop external client nodes. High connection rates.
    • Cons: Total reliance on the center unit creates a single point of structural system failure.
  • Network Layers: The intentional modular breakdown of communication processes into isolated stages.
    • Advantage: Allows updates to low-level physical channel layers (e.g., swapping an Ethernet interface to Wi-Fi) without forcing complete application tier code rewrites.

5. Essential AQA Application Protocols

Networks manage communication channels via strict, standardized software rule schemas called Protocols:

Protocol Name Functional Operational Purpose
HTTP / HTTPS Coordinates hypertext document delivery requests across web servers.
FTP Manages explicit file transfer protocols between server directories.
SMTP Handles outbound electronic mail distribution across servers.
IMAP Coordinates synchronized mail client box folder retrieval schemas.
TCP Breaks streaming content data into numbered verification packets.
IP Manages physical destination logic path routing via IP mapping.
